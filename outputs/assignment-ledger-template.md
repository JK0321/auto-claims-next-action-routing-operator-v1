## Status Options

- New
- Routed — Pending Human Approval
- Routed
- Assigned
- Pending Report
- Returned for Correction
- Complete
- Rerouted
- Closed

Use `Routed — Pending Human Approval` when a demo roster is used. This shows the operator has recommended a route and assignment mapping, but real assignment must still be approved or executed in the carrier, TPA, vendor, or claim-management system.

Use `Routed` when a routing packet has been produced without a demo roster assignment mapping.

Use `Assigned` only when real assignment has been confirmed in the carrier, TPA, vendor, or claim-management system — not from the operator alone.

---

## Optional Field Definitions

**Final disposition:** The human or system outcome after review. Examples: handled as routed, rerouted, escalated, closed, returned for follow-up. Populated after the assignment is acted on — not at routing time.

**Human override reason:** Brief explanation when a human reviewer changes or rejects the triage recommendation. Leave blank when the routing packet is accepted as-is.

---

## Usage Note

This ledger is a Markdown demo tracker. It does not assign, transmit, or manage real claims. Use it to show how routing packets could be tracked after a human or claim system approves the assignment.

Real assignment must happen in the carrier, TPA, vendor, or claim-management system.
